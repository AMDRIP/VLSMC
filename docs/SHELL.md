Здесь идеи для новых команд в Shell


Системные:
1. link - создать символическую ссылку (не поддерживается FAT16)
2. symlink - создать символическую ссылку (не поддерживается FAT16)
3. mknod - создать пустой файл
7. chmod - изменить права доступа к файлу
8. chown - изменить владельца файла
9. chgrp - изменить группу владельца файла

Работа с ядром и системой
1. crush / kill - убить процесс (kill <tid>)
2. killall - завершить все пользовательские потоки
3. fork - создать процесс
4. exec - выполнить загрузку ELF программы (exec <path>)
5. wait - дождаться завершения процесса
6. exit - завершить процесс
7. ps / threads - список активных потоков/процессов
8. clear - очистка экрана терминала
9. help - вывод справки по командам
10. ticks - вывод счетчика системных тиков
11. uptime - время работы системы
12. date - текущая дата и время (RTC)
13. sleep - приостановить выполнение (sleep <ms>)
14. yield - передать управление планировщику
15. whoami / whoiam - имя текущего пользователя

Файловая система и Удобство
1. print / echo - вывести текст на экран
2. ls - список файлов
3. cat - вывести содержимое файла на экран
4. write - записать строку в файл (write <file> <text>)
5. rm - удалить файл
6. stat - получить информацию о файле и его атрибутах
7. hexdump - вывести бинарный дамп файла
8. less / more - вывести содержимое файла на экран с постраничным выводом
9. grep / sed / awk - обработка текста (планы)

Аппаратное обеспечение и Отладка
1. pci - сканирование и вывод списка PCI устройств
2. meminfo / mems - статистика использования RAM и статус Paging
3. bootinfo - информация, переданная загрузчиком (память, видеорежим)
4. reboot - перезагрузка системы через контроллер клавиатуры 8042
5. kernelpanic - ручной вызов паники ядра
6. syscall - тест системного вызова (int 0x80)
7. ring3 - ручной запуск тестового Ring 3 потока

Тестирование (Экспериментальные)
1. memtest / pmmtest / vmmtest - тесты менеджера памяти
2. ahcitest - тест чтения/записи AHCI (ahcitest <port>)
3. sxs - запустить графическое приложение
4. extreme - запустить экстремальный режим ( opcode 0x8h )

Видео и Графика
1. mode text - переключиться в текстовый режим VGA (80x25)
2. mode gfx - переключиться в графический текстовый режим (40x25)
3. gfx - запуск VGA демо
4. bga - инициализация BGA видеодрайвера (1024x768x32)
